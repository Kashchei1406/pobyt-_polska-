# Отзывы с Google Maps на сайте

Блок «Отзывы» подгружает реальные отзывы с карточки компании в Google Maps. Для этого используется **отдельный** скрипт Google Apps Script — **не** тот, где настроены таблица и Telegram-бот. Его можно развернуть под другим аккаунтом (например, рабочим), чтобы не смешивать с личным профилем.

Файл скрипта в проекте: **`backend/ReviewsFromGoogleMaps.gs`**.

**Проект отзывов (редактор):** [script.google.com — открыть проект](https://script.google.com/home/projects/1n2owFt8qsDSdcSM0YKskOapCC2oo9hrtQDzkdsDmlO55uurwIE10s5_X/edit)

Нужно: **API-ключ**, **Place ID** и развёртывание этого скрипта. Ниже — по шагам.

Google отдаёт **не более 5 отзывов** за один запрос. Скрипт кэширует их и при каждом заходе на сайт подмешивает свежие 5 из API к кэшу (без дублей), хранит до **20 отзывов** и отдаёт их. Со временем, по мере появления новых отзывов на карточке, кэш заполняется до 20.

## Часть 1: API-ключ (Google Cloud)

### Шаг 1. Открыть Google Cloud Console

Перейдите: **https://console.cloud.google.com**

Войдите в тот аккаунт, под которым будете разворачивать скрипт отзывов (например, рабочий). Слева сверху — название проекта или «Select a project».

### Шаг 2. Выбрать или создать проект

Выберите существующий проект или **New Project** (имя, например, «Отзывы Pobyt») → **Create** → выберите его.

### Шаг 3. Включить Places API

1. Меню **☰** → **APIs & Services** → **Library**.
2. В поиске: **Places API** (именно «Places API», не «Places API (New)»).
3. Откройте карточку → **Enable**.

### Шаг 4. Создать API-ключ

1. **☰** → **APIs & Services** → **Credentials**.
2. **+ Create Credentials** → **API key**.
3. Скопируйте ключ и сохраните — это **PLACES_API_KEY**.

---

## Часть 2: Place ID (карточка компании в Maps)

1. Откройте: **https://developers.google.com/maps/documentation/places/web-service/place-id**
2. Прокрутите до **«Find place ID»**.
3. В поле поиска введите **Pobyt Polska, Warsaw** (или ссылку на карточку в Google Maps) → поиск.
4. Скопируйте **Place ID** (строка вида `ChIJ...`) — это **PLACE_ID**.

---

## Часть 3: Отдельный проект Apps Script для отзывов

Скрипт отзывов не трогает таблицы и бота — только отдаёт отзывы с Google. Его можно создать под тем же или под другим Google-аккаунтом.

### 3.1. Создать проект и вставить код

1. Откройте **https://script.google.com** и при необходимости переключитесь на нужный аккаунт.
2. **Новый проект** (или «Create» → «Project»).
3. Удалите содержимое файла `Code.gs` и вставьте **целиком** код из файла **`backend/ReviewsFromGoogleMaps.gs`** в вашем репозитории. Сохраните (Ctrl+S).

### 3.2. Script Properties (ключ и Place ID)

1. Слева нажмите **шестерёнку** (Project settings).
2. Прокрутите до **Script Properties** → **Add script property**.
3. Первое свойство: **Property** — `PLACES_API_KEY`, **Value** — вставьте API-ключ из Части 1 → **Add script property**.
4. Ещё раз **Add script property**: **Property** — `PLACE_ID`, **Value** — вставьте Place ID из Части 2 → **Add script property**.

В списке должны быть две строки: `PLACES_API_KEY` и `PLACE_ID`.

### 3.3. Развернуть как веб-приложение

1. Вверху нажмите **Развернуть** (Deploy) → **Новое развертывание** (New deployment).
2. Тип: **Веб-приложение** (Web app).
3. **Выполнять от имени**: я. **У кого есть доступ**: все (или «Anyone»).
4. **Развернуть**. Скопируйте **URL веб-приложения** (вид: `https://script.google.com/macros/s/.../exec`).

Этот URL — ваш **REVIEWS_API_URL**.

### 3.4. Указать URL на сайте

В **index.html** найдите строку:

```html
<script>window.REVIEWS_API_URL = '';</script>
```

Вставьте скопированный URL между кавычками:

```html
<script>window.REVIEWS_API_URL = 'https://script.google.com/macros/s/ВАШ_ID/exec';</script>
```

Сохраните. Обновите страницу сайта — блок «Отзывы» должен подтянуть до 5 отзывов с карточки в Google Maps.

---

## Два скрипта — два URL

| Назначение | Файл в проекте | Где разворачивать | URL в index.html |
|------------|----------------|--------------------|------------------|
| Заявки: таблица + Telegram | `backend/FormToSheetAndTelegram.gs` | Ваш личный (или любой) проект Apps Script | `window.FORM_SUBMIT_URL` |
| Отзывы с Google Maps | `backend/ReviewsFromGoogleMaps.gs` | **Отдельный** проект Apps Script (можно другой аккаунт) | `window.REVIEWS_API_URL` |

Таблица и бот остаются в одном скрипте под вашим профилем. Отзывы — в отдельном скрипте, под тем же или другим аккаунтом, без доступа к таблице и Telegram.

## Проверка: работает ли скрипт отзывов

### 1. Проверить ответ скрипта в браузере

Скопируйте **URL развёртывания** (тот же, что подставляете в `REVIEWS_API_URL`). Откройте его в новой вкладке браузера.

- **Если настроено:** увидите JSON, например `{"ok":true,"reviews":[{...},{...},...]}` — массив отзывов (до 5 штук).
- **Если не настроено:** `{"ok":false,"error":"Not configured","reviews":[],...}` — добавьте в проекте Script Properties: `PLACES_API_KEY` и `PLACE_ID`, затем сделайте новую версию развёртывания и откройте ссылку снова.
- **Другая ошибка:** в поле `error` будет указана причина (например, неверный Place ID или проблема с API-ключом).

Так вы сразу видите, отдаёт ли скрипт отзывы, без захода на сайт.

### 2. Проверить блок «Отзывы» на сайте

1. В **index.html** вставьте URL развёртывания в `window.REVIEWS_API_URL` (если ещё не вставили).
2. Откройте сайт (файлом или через локальный сервер), прокрутите до секции **«Отзывы»**.
3. Должна загрузиться карусель с карточками отзывов. Если видите «Загрузка отзывов…» и затем текст со ссылкой на Google Maps — скрипт вернул ошибку или пустой список; проверьте шаг 1 выше.

---

## Обновление настроек отзывов

Если поменяли API-ключ или Place ID в Script Properties:

1. В проекте со скриптом отзывов: **Развернуть** → **Управление развёртываниями**.
2. Рядом с развёртыванием — **карандаш** → **Версия**: **Новая версия** → **Развернуть**.

URL не меняется, в `index.html` ничего править не нужно.
