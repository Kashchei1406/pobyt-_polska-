# Как в таблице проверить скрипт

Скрипт, который принимает заявки с сайта, живёт в **Google Apps Script**, а не внутри самой таблицы. Но с таблицы можно открыть редактор скрипта и дать ему доступ к этой таблице.

---

## 1. Открыть скрипт из таблицы

1. Откройте вашу таблицу:  
   https://docs.google.com/spreadsheets/d/1fvASj36TsBrUDXObZLxCtZw5EkGrzFDAEXxKK3ruvcM/edit  
2. В меню выберите **Расширения** → **Скрипты приложений** (или **Apps Script**).  
3. Откроется проект Google Apps Script. Если проект пустой — вставьте в него код из файла **`backend/FormToSheetAndTelegram.gs`** и сохраните (Ctrl+S).

Так скрипт будет привязан к этой таблице, и при первом запуске сможет запросить доступ к ней.

---

## 2. Один раз дать доступ к таблице

Если скрипт ещё ни разу не запускали от вашего имени:

1. В редакторе Apps Script выберите в списке функций **`doPost`** (или **`doGet`**).  
2. Нажмите **Выполнить** (▶️).  
3. Появится запрос «Требуется авторизация» — нажмите **Проверить разрешения**, выберите свой Google-аккаунт, затем **Разрешить** (для доступа к таблицам и веб-приложению).  
4. После этого скрипт имеет доступ к этой таблице. Для приёма заявок с сайта вызывается `doPost`, его вручную запускать не нужно.

---

## 3. Проверить, что скрипт записывает в таблицу

**Вариант А — с сайта**  
Отправьте тестовую заявку с вашего сайта (форма с именем, телефоном, вопросом). Откройте таблицу и посмотрите, появилась ли новая строка на листе **Лиды** (или на первом листе).

**Вариант Б — тест из редактора**  
Можно добавить в проект временную функцию и запустить её вручную — она один раз добавит тестовую строку в таблицу:

1. В редакторе Apps Script в конце файла добавьте:

```javascript
function testAppendRow() {
  var spreadsheet = SpreadsheetApp.openById(SHEET_ID);
  var sheet = spreadsheet.getSheetByName(SHEET_NAME) || spreadsheet.getSheets()[0];
  sheet.appendRow([new Date(), 'Тест', '+48000000000', 'Проверка из редактора', 'test']);
}
```

2. В списке функций выберите **`testAppendRow`**, нажмите **Выполнить** (▶️).  
3. Откройте таблицу — должна появиться строка с именем «Тест» и текстом «Проверка из редактора».  
4. Когда убедитесь, что запись работает, функцию **`testAppendRow`** можно удалить.

---

## 4. Проверка по ссылке (тест без сайта)

Откройте в браузере ссылку (подставьте свой URL развёртывания):

**Ваш URL + `?test=1`**  
Например:  
`https://script.google.com/macros/s/AKfycbxADieL_vl8Etj9w_d_HY6zwcmjZ3uU2oe8oxzhIVqXVEvVrF-2SQGJxyuehZKg7H7A/exec?test=1`

Должны увидеть в ответе: `{"ok":true,"message":"Тестовая строка добавлена в таблицу..."}`. В таблице на листе «Лиды» (или первом листе) появится строка с именем «Тест GET». Если так и есть — скрипт и таблица работают, проблема в отправке формы с сайта. Если строки нет или есть ошибка — смотрите раздел 5 (логи).

---

## 5. Посмотреть логи и ошибки

1. В редакторе Apps Script слева откройте **Выполнения** (иконка с часами).  
2. Там видны последние запуски скрипта (например, при отправке формы с сайта вызывается `doPost`).  
3. Если было падение — будет красный крестик; нажмите на выполнение и посмотрите текст ошибки.  
4. В коде можно добавить **Logger.log('текст')** — тогда сообщения появятся в **Выполнения** при просмотре этого запуска.

Если заявки с сайта не приходят: откройте **Выполнения** и отправьте форму ещё раз. Появился ли вызов **doPost**? Если да — смотрите текст ошибки (красный крестик). Если вызова **doPost** нет — запрос с сайта не доходит до скрипта (проверьте URL в index.html и то, что форма реально отправляется).
