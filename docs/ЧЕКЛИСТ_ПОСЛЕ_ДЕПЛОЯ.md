# Чеклист после деплоя (формы -> Таблица + Telegram)

Используйте этот список после любых изменений в Google Apps Script, `index.html` или настройках бота.

## 1) Проверка URL веб-приложения

- В `index.html` проверьте `window.FORM_SUBMIT_URL` — должен быть актуальный URL вида `https://script.google.com/macros/s/.../exec`.
- Откройте в браузере: `FORM_SUBMIT_URL?test=1`.
- Ожидаемо: JSON с `ok: true` и тестовая строка в таблице.

Если вместо JSON страница с ошибкой доступа — deployment не публичный или открыт под не тем аккаунтом.

## 2) Проверка deployment в Apps Script

В проекте `FormToSheetAndTelegram`:

- `Deploy -> Manage deployments` (или `Развернуть -> Управление развертываниями`).
- Тип: `Web app`.
- `Execute as`: `Me`.
- `Who has access`: `Anyone`.
- После изменений кода всегда делайте `New version` и обновляйте deployment.

## 3) Проверка доступа к таблице

- Аккаунт, от имени которого выполняется web app, должен иметь минимум `Editor` к таблице с `SHEET_ID`.
- Лист существует (обычно `Лиды`) или есть первый лист в файле.
- После теста `?test=1` в таблице появляется строка `Тест GET`.

## 4) Проверка Telegram

- В скрипте не должны стоять заглушки:
  - `BOT_TOKEN != 'ВАШ_ТОКЕН_БОТА'`
  - `CHAT_ID != 'ВАШ_CHAT_ID'`
- Бот добавлен в нужный чат/группу.
- Для группы используется `chat_id` с минусом (например `-100...`).

## 5) Финальная проверка с сайта

- Отправьте тестовую форму с сайта.
- Проверьте:
  - появилась новая строка в таблице;
  - пришло сообщение в Telegram;
  - в Apps Script -> `Executions` (`Выполнения`) нет ошибок по `doPost`.

## 6) Быстрая диагностика, если снова не работает

- Откройте `FORM_SUBMIT_URL?test=1`:
  - если нет `ok: true`, проблема в deployment/доступе;
  - если `ok: true`, но с сайта не пишет — проверьте `FORM_SUBMIT_URL` в `index.html` и кеш браузера.
- Сделайте hard refresh страницы (`Ctrl+Shift+R`).
- При необходимости включите отладку в браузере:
  - `localStorage.setItem('debugForm','1')`
  - обновите страницу и отправьте заявку еще раз.
