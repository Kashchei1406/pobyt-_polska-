# POBYT POLSKA — лендинг

Файлы:
- `index.html` — разметка
- `styles.css` — стили
- `main.js` — интерактив (FAQ, квиз, модалки, отправка форм, cookie)
- `backend/FormToSheetAndTelegram.gs` — скрипт для приёма заявок в Google Таблицу и в Telegram
- `backend/ReviewsFromGoogleMaps.gs` — отдельный скрипт только для отзывов с Google Maps (можно развернуть под другим аккаунтом)

## Как запустить

Самый простой способ — открыть `index.html` в браузере.

Если нужен локальный сервер (рекомендуется для работы форм):

```bash
python3 -m http.server 5173
```

Затем открыть: `http://localhost:5173`

### Запуск на телефоне в одной Wi‑Fi сети

1. На компьютере в папке проекта запустите сервер так, чтобы он принимал подключения из сети:
   ```bash
   python3 -m http.server 5173 --bind 0.0.0.0
   ```
2. Узнайте IP-адрес компьютера в локальной сети:
   - **macOS / Linux:** в терминале `ifconfig | grep "inet "` или `ip addr` — нужен адрес вида `192.168.x.x` (не 127.0.0.1).
   - **Windows:** `ipconfig` — смотрите «IPv4-адрес» для вашего Wi‑Fi.
3. Телефон подключите к **той же Wi‑Fi сети**, что и компьютер.
4. В браузере на телефоне откройте: `http://192.168.x.x:5173` (подставьте свой IP вместо `192.168.x.x`).

Сайт откроется на телефоне. Формы и заявки работают так же.

## Сбор заявок: таблица + Telegram

Заявки с форм (имя, телефон, вопрос) сохраняются в **Google Таблицу** и при настройке — дублируются в **чат Telegram-бота**. Пошагово: [docs/НАСТРОЙКА_ТЕЛЕГРАМ.md](docs/НАСТРОЙКА_ТЕЛЕГРАМ.md).

### 1. Google Таблица

Таблица уже настроена (ID подставлен в скрипт). **Что сделать в самой таблице** — по шагам: см. файл [docs/ШАГИ_В_ТАБЛИЦЕ.md](docs/ШАГИ_В_ТАБЛИЦЕ.md).

Кратко: откройте таблицу → переименуйте лист в **Лиды** → в первой строке введите заголовки: **Дата** | **Имя** | **Телефон** | **Город** | **Вопрос** | **Источник**. (Если таблица уже была без столбца «Город», вставьте новый столбец между «Телефон» и «Вопрос» и обновите скрипт в Apps Script из `backend/FormToSheetAndTelegram.gs`.)

### 2. Telegram-бот

1. В Telegram откройте [@BotFather](https://t.me/BotFather), отправьте `/newbot` и следуйте подсказкам.
2. Скопируйте выданный **токен** бота (например, `123456:ABC-DEF...`).
3. Узнайте **chat_id**: напишите боту любое сообщение, затем откройте в браузере (подставьте свой токен):  
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates`  
   В ответе найдите `"chat":{"id": 123456789}` — это ваш `chat_id`.

### 3. Google Apps Script

1. Откройте [script.google.com](https://script.google.com) и создайте новый проект.
2. Вставьте код из файла `backend/FormToSheetAndTelegram.gs`.
3. В начале скрипта замените:
   - `BOT_TOKEN` — токен бота из BotFather (для теста таблицы можно оставить как есть, в Telegram ничего не уйдёт)
   - `CHAT_ID` — ваш chat_id (для теста таблицы можно оставить как есть)
   - `SHEET_ID` — уже подставлен (ваша таблица). Лист должен называться **Лиды**: внизу таблицы переименуйте «Sheet1» в «Лиды» или в скрипте задайте `SHEET_NAME = 'Sheet1'`.
4. В первой строке этого листа введите заголовки: **Дата** | **Имя** | **Телефон** | **Вопрос** | **Источник**.
5. Сохраните проект (Ctrl+S). Разверните: **Развернуть** → **Новое развертывание** → тип **Веб-приложение**:
   - **Выполнять от имени**: я
   - **У кого есть доступ**: все
6. Нажмите **Развернуть**, скопируйте **URL веб-приложения** (вид: `https://script.google.com/macros/s/.../exec`).

**Проверка таблицы:** можно не заполнять Telegram (оставьте `BOT_TOKEN` и `CHAT_ID` как есть) — скрипт будет только записывать заявки в таблицу. После настройки Telegram заявки начнут дублироваться в чат.

### 4. Подключение URL на сайте

В `index.html` найдите строку:

```html
<script>window.FORM_SUBMIT_URL = '';</script>
```

Вставьте скопированный URL между кавычками:

```html
<script>window.FORM_SUBMIT_URL = 'https://script.google.com/macros/s/ВАШ_ID/exec';</script>
```

Сохраните файл. После этого при отправке любой формы с сайта заявка будет добавляться в таблицу и приходить в Telegram.

Если `FORM_SUBMIT_URL` оставить пустым, формы будут работать по-старому: показывать сообщение «Готово! Мы получили заявку» без реальной отправки.

**Если в консоли виден 403 на `script.googleusercontent.com`:** заявки уходят POST-формой в скрытый iframe (так надёжнее для Google Apps Script); 403 часто относится к редиректу ответа, а не к записи в таблицу — проверьте лист **Лиды** и раздел **Выполнения** в редакторе Apps Script. Для отладки в консоли: `localStorage.setItem('debugForm','1')` и снова отправить форму — в консоли появится URL и поля. Обновите страницу с жёстким сбросом кэша (Ctrl+Shift+R). Развёртывание: **У кого доступ — Все (Anyone)** и после правок кода — **Новая версия**.

### 5. Отзывы с Google Maps (опционально)

Блок «Отзывы» подгружает реальные отзывы с карточки в Google Maps через **отдельный** скрипт — `backend/ReviewsFromGoogleMaps.gs`. Его разворачивают в **другом** проекте на script.google.com (можно под другим аккаунтом, чтобы не смешивать с таблицей и ботом). В `index.html` задаётся **`window.REVIEWS_API_URL`** — URL развёртывания этого скрипта. Подробно: [docs/ОТЗЫВЫ_GOOGLE_MAPS.md](docs/ОТЗЫВЫ_GOOGLE_MAPS.md). Если `REVIEWS_API_URL` пустой, в блоке показывается текст со ссылкой на карточку в Google Maps.

### Источники заявок

В таблице и в Telegram в колонке/строке **Источник** указывается, откуда пришла заявка:

- `hero` — форма в первом экране
- `contacts` — форма в блоке «Контакты»
- `header` / `mobile` / `services` — форма из модального окна (кнопка в шапке, в меню, в блоке услуг)
